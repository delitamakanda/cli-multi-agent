# Codebase Guidelines

When onboarding new developers to this Python repository, please follow these guidelines:

## Project Structure
- Familiarize yourself with the project structure and organization.
- Understand the purpose of each module and package within the codebase.
- Identify the main entry points and how different components interact with each other.
- Review the README.md and any other documentation to understand the project's goals and functionality.
- Explore the directory structure and identify any submodules, external dependencies, or third-party libraries used in the project.
- Understand the build and deployment process, including any scripts or tools used for automation.
- Review the version control system (e.g., Git) and understand the branching strategy, commit conventions, and pull request workflow.
- Familiarize yourself with any coding standards or style guides followed by the team, such as PEP 8 for Python code.
- Review any existing tests and understand the testing framework and coverage requirements.
- Understand the project's configuration files, such as .env, config.yaml, or any other relevant files that define environment variables or settings.
