When generating commit messages for code changes in this python repository, please follow these guidelines:

## Commit Message Structure
- Use the imperative mood in the subject line (e.g., "Fix bug" instead of "Fixed bug" or "Fixes bug").
- Limit the subject line to 50 characters or less.
- Capitalize the first letter of the subject line.
- Do not end the subject line with a period.
- Separate the subject line from the body with a blank line.
- Use the body to explain what and why vs. how. The body should be wrapped at 72 characters.
- Use bullet points or lists in the body if necessary to explain multiple changes or reasons.

## Commit Message Content
- Clearly describe the changes made in the commit.
- Include relevant issue or ticket numbers in the commit message if applicable.
- Avoid vague or generic commit messages (e.g., "Update code" or "Fix stuff").
- Use consistent terminology and phrasing throughout the commit messages.
- Avoid using personal pronouns (e.g., "I", "we") in the commit message.
- Ensure that the commit message is understandable to someone who is not familiar with the codebase.
- Use the body of the commit message to provide context and reasoning for the changes made.
- Avoid including sensitive information or secrets in the commit message.
- Use the body of the commit message to explain any trade-offs or design decisions made during the implementation of the changes.
- Use the body of the commit message to provide any relevant references or links to documentation, code reviews, or related commits.
- Use the body of the commit message to explain any potential impact or side effects of the changes made.
